## 修課心得

在大二開始前，因為本身不太擅長程式，又是轉系生，讓我感到非常焦慮。我不僅擔心自己跟不上同學的進度，也害怕上課時聽不懂老師的講解。當時打開課表，發現與程式最相關的科目就是「資料結構」，
因此這門課成為我最擔心的一科。剛開學時，老師在 portal 上公告需要會使用 GitHub，但那時的我甚至不知道 GitHub 是什麼，只覺得聽起來很困難，因此又更焦慮。

真正開始上資料結構課後，我的想法逐漸改變。一開始老師將重點放在讓我們了解「什麼是資料結構」，這其實也是我在開學前曾經上網查過、卻沒有完全理解的問題。接著，課程從最基本的 array 開始，
再依序介紹 linked list、stack、queue 等主題。這樣的上課方式非常適合我，老師講解清楚、節奏不快，讓我能夠一步一步吸收內容。若是採用其他的上課順序和方式，我想自己會感到相當混亂，也需要花更多時間才能理解。

在作業方面，有些內容會搭配 AI 工具完成。透過這樣的方式，我開始思考該如何下清楚且完整的指令，也因此體會到善用 AI 來整理與理解資訊的重要性。此外，使用 GitHub 來記錄課堂重點，
我也認為這是一個很好的整理方法。雖然一開始需要花較多時間查詢如何上傳圖片、調整字體大小等，但在多次嘗試後便逐漸熟悉，也提升了我使用工具的能力。

總體而言，資料結構這門課不僅穩固了我的程式基礎，也讓我學到許多額外的技能，例如用程式畫出不同的樹狀結構、使用 GitHub 進行整理，以及透過 AI 協助解決問題，讓我在這門課中受益良多。
